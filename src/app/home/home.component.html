Realtime Rendered Résumé

This site contains a component to render my personal résumé as a web page using live information from my LinkedIn profile. This is accomplished using another project, written in Golang, that provides a simple API to retrieve my Linkedin profile as JSON. This it allows myself, or anyone else, to view a formatted résumé that is always up-to-date with my LinkedIn profile.

Fully Automated Build, Test, Deploy

The Angular project backing this site has fully-automated CI/CD using GitHub Actions. The steps are defined in a workflows file contained within the project's source and are executed each time there is a push to the main branch of the project. This means that any change to the project that is pushed or merged onto the project's default git branch is automatically executed in the CI/CD pipeline and the application is re-deployed into is hosting environment.

In addition, the Golang project providing the Linkedin profile API also has fully-automated CI/CD. But instead of Github Actions, this project is using Google Cloud Build to automatically pickup any commits to the Github repository and perform CI/CD automation. This includes the construction of a Docker image for the Golang project and the deployment of this image into Google Cloud Run for hosting.

As a developer, these CI/CD automations make the development process so much smoother than if the process to deliver the code to production was manual. For my small project, this isn't specatular, but on a large project with many team members, this is crucial to establishing agility in the software delivery process. Teams get continuous feedback on their own changes as well as others', test failures can be readily identified and corrected, and the team is able to stay focused on code instead of the act of getting it into its execution environment.

Hosting of this site

The Angular application for this site is hosted as a static website within an AWS (Amazon Web Services) S3 bucket. The bucket holds the site's constructed Angular output files, while an AWS CloudFront distribution is setup to serve the actual files via HTTP requests, as well as providing Transport Layer Security (TLS) for the site.

For the Golang project providing the Linkedin profile data for myself, it is deployed as a Docker container into Google Cloud Run
